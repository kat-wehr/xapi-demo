Design Notes & Implementation Guidance
  • All activity IRIs should follow a consistent namespace: https://[institution].org/activities/[course]/microcritique/[step]
  • The issue_type extension vocabulary should be defined in a shared xAPI Profile document and reused across all weekly modules for cohort-level analytics
  • Response text in the result.response field enables AI-assisted rubric scoring or NLP analysis downstream — ensure your LRS data retention policy covers PII in free-text fields
  • Use context.contextActivities.parent to link each step to the parent assignment, and grouping to link to the module — this enables roll-up reporting across weeks
  • MC-041 uses a StatementRef as the object so the score is permanently linked to the learner's original submission statement in the LRS
  • Consider adding a revised verb statement if the platform supports iterative drafts — tracking revision behavior can be a valuable leading indicator of skill growth
